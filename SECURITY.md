# 安全策略

mikuOJ 的核心职责是安全地编译和运行不受信任的用户代码。任何沙箱逃逸、资源限制绕过或敏感信息泄露都属于高优先级安全问题。

## 支持范围

当前正式 OJ 后端为 Linux `linux-ns`：

- namespace 隔离
- cgroup v2 资源限制
- seccomp 默认拒绝白名单
- 降权到 `nobody`
- 最小文件系统白名单

## 报告漏洞

请不要在公开 issue 中披露可利用的沙箱逃逸或隔离绕过细节。

报告时请附：

- 受影响后端：正式 OJ 后端为 `linux-ns`
- 内核版本和运行环境
- 复现提交代码 PoC
- 观察到的行为
- 预期行为

## 威胁模型

可信对象：

- 判题系统本体
- 编译器 / 解释器工具链
- 题目数据

不可信对象：

- 用户提交的源代码
- 编译产物
- 用户程序运行时行为
- 用户程序输出

## 已知限制

- 编译阶段的工具链 syscall 面较大，安全性依赖 namespace、cgroup、降权和最小文件系统隔离。
- 当前未使用 user namespace，后续可继续强化 UID/GID 映射模型。
- 需要 root / delegated cgroup 环境才能完整使用 Linux 安全后端。
